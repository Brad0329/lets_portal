# LETS 프로젝트 관리 시스템 — 설치 및 사용 매뉴얼

## 1. 준비물

| 항목 | 설명 |
|------|------|
| 서버 PC | Windows 10/11, 항상 켜져 있어야 함 |
| 납품 폴더 | USB 또는 공유폴더로 전달받은 `LETS_DEPLOY` 폴더 |
| API 키 | 공공데이터포털(data.go.kr) API 키 (별도 전달) |

## 2. 서버 PC 설정 (관리자 1회)

### 2-1. 폴더 복사

납품받은 `LETS_DEPLOY` 폴더를 서버 PC의 원하는 위치에 복사합니다.

```
예시: C:\LETS_DEPLOY\
```

폴더 구조:
```
LETS_DEPLOY\
├── lets_server.exe         ← 서버 실행 파일
├── .env                    ← API 키 설정
├── frontend\               ← 화면 파일
├── data\                   ← DB (첫 실행 시 자동 생성)
└── collectors\
    └── scraper_configs.json
```

### 2-2. API 키 설정

`.env` 파일을 메모장으로 열어 API 키를 입력합니다.

```
DATA_GO_KR_KEY=발급받은API키를여기에입력
```

> API 키는 https://www.data.go.kr 에서 회원가입 후 발급받을 수 있습니다.
> "나라장터 입찰공고정보서비스", "K-Startup 조회서비스" 활용신청이 필요합니다.

### 2-3. 서버 실행

`lets_server.exe`를 더블클릭합니다.

```
검은 콘솔 창이 열리며 아래 메시지가 표시되면 정상:

INFO:     Started server process [12345]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

> **주의:** 콘솔 창을 닫으면 서버가 종료됩니다. 최소화 상태로 유지하세요.

### 2-4. Windows 방화벽 허용

첫 실행 시 "Windows 보안 경고" 팝업이 나타납니다.

1. **"개인 네트워크"에 체크**
2. **"액세스 허용"** 클릭

팝업이 안 나타난 경우 수동 설정:
1. Windows 검색 → "방화벽" → "Windows Defender 방화벽"
2. 좌측 "인바운드 규칙" → "새 규칙"
3. 포트 → TCP → 특정 로컬 포트: `8000`
4. 연결 허용 → 이름: `LETS 서버` → 마침

### 2-5. 서버 PC의 IP 주소 확인

1. `Win + R` → `cmd` 입력 → Enter
2. `ipconfig` 입력 → Enter
3. **"IPv4 주소"** 확인 (예: `192.168.0.10`)

```
무선 LAN 어댑터 Wi-Fi:
   IPv4 주소 . . . . . . . . : 192.168.0.10    ← 이 주소를 직원들에게 공유
```

> 이 IP 주소를 직원들에게 알려주세요.

### 2-6. 관리자 계정 최초 로그인

첫 실행 시 기본 관리자 계정:

| 항목 | 값 |
|------|-----|
| ID | `admin` |
| 비밀번호 | `1234` |

1. 브라우저에서 `http://192.168.0.10:8000` 접속
2. `admin` / `1234` 로 로그인
3. **비밀번호 변경 화면이 나타나면 반드시 변경**
4. 설정 → 계정 관리에서 실무자 계정 생성

## 3. 실무자 접속 방법

### 3-1. 접속

1. 크롬, 엣지 등 웹 브라우저를 엽니다
2. 주소창에 관리자가 알려준 주소를 입력합니다

```
http://192.168.0.10:8000
```

> 서버 PC와 같은 공유기(네트워크)에 연결되어 있어야 합니다.
> Wi-Fi, 유선 모두 가능합니다.

### 3-2. 로그인

- 관리자가 생성해준 ID와 초기 비밀번호(`1234`)로 로그인
- 첫 로그인 시 비밀번호 변경 필수

### 3-3. 즐겨찾기 등록 (권장)

매번 주소를 입력하지 않도록 즐겨찾기에 추가하세요.

- **크롬:** 주소창 오른쪽 별표 아이콘 클릭
- **엣지:** 주소창 오른쪽 별표 아이콘 클릭

## 5. 서버 관리

### 서버 시작

`lets_server.exe` 더블클릭

### 서버 종료

콘솔 창에서 `Ctrl + C` 또는 콘솔 창 닫기

### PC 재시작 후

서버가 자동 실행되지 않으므로, PC 재시작 후 `lets_server.exe`를 다시 실행해야 합니다.

#### 자동 시작 설정 (선택)

PC 부팅 시 자동으로 서버를 시작하려면:

1. `Win + R` → `shell:startup` 입력 → Enter
2. 열린 폴더에 `lets_server.exe`의 **바로가기**를 복사
3. 이후 PC 시작 시 자동으로 서버가 실행됩니다

### 데이터 백업

`data\portal.db` 파일을 복사하면 전체 데이터가 백업됩니다.

```
예시: data\portal.db → D:\백업\portal_20260402.db
```

### 데이터 이관 (서버 PC 변경 시)

1. 기존 서버 PC에서 `LETS_DEPLOY` 폴더 전체를 복사
2. 새 서버 PC에 붙여넣기
3. `lets_server.exe` 실행
4. 새 서버 PC의 IP 주소를 직원들에게 공유

## 6. 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 접속 안 됨 | 서버가 꺼져 있음 | 서버 PC에서 `lets_server.exe` 실행 확인 |
| 접속 안 됨 | 방화벽 차단 | 2-4 방화벽 허용 설정 확인 |
| 접속 안 됨 | IP 주소 변경됨 | 서버 PC에서 `ipconfig`로 새 IP 확인 후 공유 |
| 접속 안 됨 | 다른 네트워크 | 같은 공유기에 연결되어 있는지 확인 |
| 서버 실행 시 오류 | 포트 충돌 | 다른 프로그램이 8000번 포트를 사용 중. 해당 프로그램 종료 |
| 로그인 안 됨 | 비밀번호 분실 | 관리자에게 비밀번호 초기화 요청 |
| 수집 안 됨 | API 키 오류 | `.env` 파일의 API 키 확인 |
| 백신이 차단 | PyInstaller exe 오탐 | 백신에서 `lets_server.exe` 예외 등록 |

### IP 고정 설정 (권장)

서버 PC의 IP가 바뀌면 직원들이 접속할 수 없습니다. IP를 고정하려면:

1. Windows 설정 → 네트워크 → 이더넷 (또는 Wi-Fi) → IP 설정 편집
2. 수동으로 변경
3. IP 주소: 현재 IP (예: `192.168.0.10`)
4. 서브넷 마스크: `255.255.255.0`
5. 게이트웨이: 공유기 IP (보통 `192.168.0.1`)
6. DNS: `8.8.8.8`
