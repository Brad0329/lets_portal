# Phase 2 작업 로그

## 기간
2026-03-27

## 목표
설정 페이지 + UI 고도화 + 자동 수집 스케줄러

---

## 완료 항목

### 2-1: 설정 페이지 (`/settings.html`)
- 표시 설정: 상태 필터, 조회 기간, 정렬 기준, 페이지당 건수
- 키워드 관리: 그룹별 칩 UI, 클릭으로 ON/OFF 토글, 전체 ON/OFF
- 기관 목록: 58개 기관 테이블 (링크, 분류, 활성 상태)
- 수집 실행: 설정 변경 후 즉시 수집 가능
- 설정 저장 API 연동 (PUT /api/settings/{key})

### 2-2: 프론트엔드 UI 고도화
- **상세보기 모달**: 행 클릭 시 공고 상세 정보 팝업
  - 출처, 기관, 분류, 공고번호, 시작/마감일, 상태, 키워드
  - "공고 사이트 바로가기" 버튼
  - ESC 키 / 배경 클릭으로 닫기
- **헤더 개선**: 설정 페이지 링크 버튼, flex 레이아웃
- **반응형**: 모바일/데스크탑 대응

### 2-3: 자동 수집 스케줄러
- APScheduler BackgroundScheduler 적용
- 매일 09:00, 14:00 자동 수집 (cron)
- FastAPI lifespan 이벤트에서 시작/종료 관리
- GET /api/scheduler 상태 조회 API

### 2-4: 라우팅 수정
- settings.html 명시적 FileResponse 라우트 추가
- StaticFiles html=True와 공존하도록 처리
