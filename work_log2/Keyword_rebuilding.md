# 키워드 리빌딩

## 1. 목적

나라장터 수집 성능 개선을 위한 키워드 정리.

- 나라장터 API는 키워드별 × 입찰유형별(3) × 날짜구간별로 호출하므로, 키워드 수가 곧 API 호출 횟수에 직결
- 85개 키워드 × 3 입찰유형 = 최소 255회 API 호출, 매 호출 0.5초 대기 → 최소 2분+
- 중복/포함 관계인 키워드를 정리하여 불필요한 API 호출 제거

| 삭제 키워드 | 유지 키워드 | 사유 |
|------------|-----------|------|
| `start up` | `startup` | 띄어쓰기 차이. "startup"으로 충분 |
| `로컬 크리에이터` | `로컬크리에이터` | 띄어쓰기 차이. 붙여쓴 형태로 통일 |
| `상권 활성화` | `상권`, `활성화` | 띄어쓰기 차이이며, "상권"과 "활성화" 두 키워드로 이미 커버 |
